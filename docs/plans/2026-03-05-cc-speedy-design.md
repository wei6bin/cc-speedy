# cc-speedy Design

Date: 2026-03-05

## Overview

A terminal-based Claude Code session launcher. Shows recent conversations across all projects, lets you resume them in named tmux sessions, and auto-generates summaries via a SessionEnd hook.

## Goals

- Quick entry point from the command line to browse and resume Claude Code sessions
- Named tmux sessions per project so sessions persist and are reachable by name
- Conversation summaries auto-generated at session end, with on-demand fallback in the TUI

## Architecture

Single Rust binary with three subcommands:

```
cc-speedy              # default: launch TUI
cc-speedy summarize    # called by SessionEnd hook
cc-speedy install      # patches ~/.claude/settings.json to register the hook
```

### Data Flow

```
~/.claude/projects/*/**.jsonl  ──► cc-speedy (TUI)  ──► session list
~/.claude/summaries/<id>.md   ──┘

SessionEnd hook ──► cc-speedy summarize <session-id> <project-path>
                       └──► Claude API (haiku-4-5) ──► ~/.claude/summaries/<id>.md

TUI select ──► tmux new-session -s <project-name> -c <project-path>
                └──► claude --resume <session-id>
```

Key bindings:
- `j/k` or arrows: navigate list
- `/`: fuzzy filter by project name
- `Enter`: resume session in tmux
- `r`: force-regenerate summary for selected session
- `q`: quit

## Tmux Workflow

When Enter is pressed on a session:
1. Derive session name from last 2 path segments of project path (e.g. `ai-zero-drift-chat`)
2. If tmux session with that name already exists: `tmux switch-client -t <name>`
3. Otherwise: `tmux new-session -d -s <name> -c <project-path> "claude --resume <session-id>"` then `tmux switch-client -t <name>`
4. If not inside tmux: `tmux new-session -s <name> -c <project-path> "claude --resume <session-id>"`

## Summary Generation

### Path A: SessionEnd Hook (automatic)

`cc-speedy install` adds to `~/.claude/settings.json`:
```json
"SessionEnd": [{
  "hooks": [{
    "type": "command",
    "command": "cc-speedy summarize"
  }]
}]
```

The hook reads `CLAUDE_SESSION_ID` and `CLAUDE_PROJECT_DIR` env vars (provided by Claude Code), calls the Anthropic API with the last 50 messages, and writes `~/.claude/summaries/<session-id>.md`.

### Path B: On-demand in TUI (fallback)

When a session is selected and no summary file exists, an async task is spawned to generate it. A spinner is shown in the preview pane. Press `r` to force-regenerate.

### Summary Format

```markdown
## What was done
- Short bullet points of the conversation topic

## Files changed
- List of key files touched (if any)

## Status
Completed / In progress / Abandoned
```

Generated by claude-haiku-4-5 with a concise prompt targeting 3-5 bullets.

## Session Filtering

Sessions are excluded from the list if:
- Fewer than 4 messages (command-line only or empty)
- Only contain `local-command-caveat` wrapper with no real content

Sessions are sorted by file modification time, most recent first.
